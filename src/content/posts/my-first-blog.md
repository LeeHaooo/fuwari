## 🐙 第一次接触 GitHub

![GitHub学习](/images/first2.png)

一开始，我对 GitHub 的理解很简单：

```bash
git clone
```

这条命令可以把项目下载到本地。
但真的把项目下载下来以后，我才发现：    

> 我会 Clone，却不会运行项目。

于是开始学习：

* README 的作用
* 什么是依赖
* package.json 是什么
* npm 与 pnpm 的区别
* Astro 是什么

这些概念刚接触时有点乱。看多了、试多了以后，我才慢慢明白，一个项目从下载到运行，中间还有不少步骤。慢慢理解了一个项目从下载到运行的完整流程。

---

## 💻 搭建博客

为了拥有自己的博客，我选择了 Fuwari 主题。

### 安装依赖

Python项目：        

```bash
pip install
```

前端项目：

```bash
pnpm install
```

虽然工具不同，但本质都是安装项目运行所需的依赖。

---

### 启动项目

```bash
pnpm dev
```

启动成功后访问：

```text
http://localhost:4321
```

第一次看到博客真的跑起来时，我盯着页面看了好一会儿。那种感觉很直接：原来我也能把一个项目跑出来。

---

### 个性化博客

## 💡 为什么写博客

### 记录学习过程

很多知识当时觉得懂了，过几天就会忘。

写博客能逼着我把思路整理清楚，也方便以后查回来。

---

### 沉淀知识

如果我能把一个知识点写成文章，说明我至少已经试着把它讲明白了。
这个过程会暴露很多模糊的地方。写到一半发现自己说不清，往往就是还没真正理解。

---

### 见证成长

几年后再回头看这篇文章时，也许会发现：

* 当年的自己刚学 Python
* 刚接触 GitHub
* 刚拥有第一个博客

那时的我可能已经在写更复杂的项目。也可能还在慢慢补基础。不管怎样，这篇文章都算是一个起点。
